請注意，所有新增或修改的功能必須嚴格遵循「零破壞原則」，確保不會干擾現有的 ApexCharts K 線圖及 blended valuation 計算模組的運行穩定性與視覺完整性。

### 📊 功能需求分析與實作建議

#### 1. 首頁英文名稱按鈕 (Homepage English Name Button)
*   **類型：** UI/UX 導航新增。
*   **架構考量：** 此按鈕應作為一個獨立的 `NavLink` 組件，放置在現有導航欄（Navbar）結構中。點擊事件需觸發路由切換至「總體經濟分析頁面」。
*   **實作建議：** 確保該組件的樣式設計與既有的按鈕系統保持一致性，避免引入新的 CSS Scope Conflict。

#### 2. 總體經濟頁面國家選擇器 (Macroeconomic Page Country Selector)
*   **類型：** 功能切換、狀態管理。
*   **架構考量：** 需要在頁面頂部新增一個國家選擇下拉式選單（Dropdown/Toggle Switch）。此選擇器的值必須作為核心的 **State Variable**，並驅動後端 API 呼叫參數。
    *   當使用者切換國家時，應觸發數據 Hook (e.g., `useEffect` 或 Vue 的 `watch`)，重新獲取該國家的所有相關指標數據。
    *   必須設計一個 **Loading State** 和 **Error State** 來處理 API 呼叫的延遲或失敗情況，避免介面卡死。
*   **實作建議：** 將國家選擇器與圖表組件進行解耦（Decouple）。即：選擇器負責提供參數 $\rightarrow$ 數據層負責獲取數據 $\rightarrow$ 圖表組件只負責渲染接收到的數據。

#### 3. 圖表雙縱座標配置 (Dual Y-Axis Charting)
*   **類型：** 圖表渲染邏輯調整。
*   **架構考量：** 這需要修改圖表庫（ApexCharts）的配置物件結構，將原本單一的 `yaxis` 配置擴展為兩個獨立的軸 (`yaxis[0]` 和 `yaxis[1]`)。
    *   **數據映射 (Data Mapping)：** 必須明確定義哪一個指標（例如：指數變化率）應綁定到主軸 $Y_1$，另一個指標（例如：利率變動）應綁定到次軸 $Y_2$。
    *   **比例尺調整 (Scale Management)：** 由於兩個指標的數值範圍可能差異極大，必須在配置中加入 `opposite: true` 和獨立的 `labels` 配置，確保兩個軸的刻度線和標籤不會互相干擾或視覺上產生誤導。
*   **風險提示：** 此修改會增加圖表渲染的複雜性，需進行單元測試 (Unit Test) 驗證不同數值範圍下的顯示效果。

#### 4. 圖表字型大小增大 (Chart Font Size Increase)
*   **類型：** UI/UX 樣式調整。
*   **架構考量：** 這屬於全局的 CSS 或圖表組件層級的樣式覆蓋（Style Overriding）。建議在圖表容器的根元素上，定義一個新的 CSS Class (e.g., `macro-chart-large`)，並將字體大小提升 2 個單位。
*   **實作建議：** 為確保修改的局部性，應使用 CSS Modules 或 Styled Components 等方式進行樣式隔離（Scope Isolation），避免影響到其他非圖表區域的文字顯示。

### 🛡️ 開發流程與專案防禦要點總結

在實際開發過程中，請務必注意以下三個關鍵的「踩坑防禦要點」：

1.  **【零破壞原則】（核心）：**
    *   所有新增的國家選擇器、按鈕或圖表修改，都必須作為獨立的組件 (Component) 進行開發與測試。
    *   在整合階段，需特別驗證 K 線圖和 blended valuation 的數據流是否被新的狀態管理邏輯意外干擾。

2.  **【JSON 序列化保護】（資料層）：**
    *   由於總體經濟分析涉及多種時間序列數據，當從後端 API 接收到包含日期或浮點數的結構化數據時，在將其傳遞給前端狀態管理或圖表渲染前，必須執行嚴格的 **Data Transformation Layer**。
    *   請確保所有 `datetime` 和 `Decimal` 類型都已轉換為標準的 ISO 8601 字串格式 (`YYYY-MM-DD HH:MM:SS`) 或浮點數 (`float`)。

3.  **【Unicode 防禦】（檔案 I/O）：**
    *   如果未來在總體經濟頁面新增任何需要讀取本地 CSV、Excel 或配置文件的功能，請務必使用 `open(..., encoding='utf-8', errors='ignore')` 語法來處理文件流，防止編碼衝突導致的解碼錯誤。
