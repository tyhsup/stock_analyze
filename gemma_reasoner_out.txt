#### 2. 資料模型與 API 層更新 (`models/macro_us.py` & `api/v1/macro_service.py`)
我們需要擴充資料庫的欄位結構，並新增計算邏輯。

**A. 資料庫 Schema 更新 (MySQL):**
在 `macro_us` 表中加入新的欄位（例如：`treasury_yield`, `aaa_bond_rate`, `core_pce`, etc.）。

**B. 數據傳輸保護 (JSON Serialization Protection)：**
當從 MySQL 查詢資料集並準備回傳給 API 層時，必須確保所有非標準 Python 類型（如 `datetime` 或 `Decimal`）都被轉換為 JSON 可接受的格式。

```python
# 【專案防禦要點：JSON 序列化保護】實作範例
def serialize_macro_data(db_record):
    """將包含日期和精確數值的資料記錄，安全地轉成字典結構。"""
    safe_data = {}
    for key, value in db_record.items():
        if isinstance(value, (datetime.date, datetime.datetime)):
            # 轉換為標準 ISO 格式字串
            safe_data[key] = value.isoformat()
        elif isinstance(value, decimal.Decimal):
            # 轉換為浮點數字串，避免 JSON 庫無法處理 Decimal 物件
            safe_data[key] = float(value)
        else:
            safe_data[key] = value
    return safe_data

# API 層呼叫時必須使用此函數進行包裹。
```

**C. 衍生指標計算邏輯：**
在 `MacroService` 中，新增一個方法來計算信用利差等指標，確保數據的原子性計算，而非依賴前端計算。
